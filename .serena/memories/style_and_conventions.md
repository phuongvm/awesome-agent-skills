# Code Style and Conventions for Awesome Agent Skills

## Documentation Standards
- Markdown format for all documentation
- Multi-language support (English, Chinese, Japanese, Korean, Spanish)
- Clear categorization of skills
- Links to original sources
- Verification before adding skills

## Skill Listing Format
- Organized by category
- Include description and source
- Link to original repository
- Note compatibility/platform support

## Contribution Guidelines
- Follow skill template structure
- Provide clear, actionable instructions
- Include working examples
- Document trade-offs and potential issues
- Keep SKILL.md under 500 lines
- Verify skills exist before adding

## Best Practices
- Maintain accurate links
- Keep categories organized
- Update regularly with new skills
- Verify skill compatibility
- Provide clear usage instructions
